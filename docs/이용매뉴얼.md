# 공공저작물 권리유형 자동분류 서비스 — 이용 매뉴얼

> 본 매뉴얼은 서비스의 **사용 방법**과 **결과 화면을 보는 방법**을 안내한다.
> 서비스 주소: https://kogl-classifier.vercel.app

---

## 1. 서비스 개요

계약서·저작물 파일을 업로드하면 **공공누리 유형(제1~4유형)** 과 **저작권 권리 판정(허용/금지)** 을 자동으로 분류·추정하고, 그 **판정 근거를 원문에서** 확인할 수 있는 서비스다.

| 메뉴 | 무엇을 하나 | 결과로 무엇을 보나 |
|---|---|---|
| **검사하기** | 계약서·저작물로 공공누리 유형 분류 | 공공누리 유형 + 저작물 메타데이터 |
| **권리추정** | 계약서에서 권리 항목별 허용/금지 추정 | 권리 판정표 + 근거 |
| **통합검사** | 위 둘을 한 화면에서 제공 | 메타데이터 + 유형 + 권리 + 근거 하이라이트 |

---

## 2. 시작하기

1. 브라우저에서 https://kogl-classifier.vercel.app 접속
2. **이메일 · 비밀번호** 입력 후 **[로그인]**
3. 좌측 사이드바에서 **검사하기 · 권리추정 · 통합검사** 중 원하는 메뉴 선택

---

## 3. 기본 사용 흐름 (공통)

세 메뉴 모두 사용법은 동일하고 간단하다.

1. **목록 화면**에서 첫 번째 **[새 검사 시작]** 카드를 누른다.
2. **파일을 업로드**하고(권리추정은 텍스트 붙여넣기도 가능) 검사 명칭을 입력한다.
3. **[실행]** 을 누르면 자동으로 처리된다.
4. 처리가 끝나면 카드 또는 상세 화면에서 **결과를 확인**한다.

### 처리 진행 단계

실행하면 아래 단계로 진행되며, 상세 화면에 **실시간으로** 표시된다.

```
업로드 → OCR·메타데이터 → (공공누리 유형) → 권리 판정 → 완료
```

- 처리 중에는 화면이 자동으로 갱신된다(별도 새로고침 불필요).
- 문서 크기에 따라 수십 초~수 분 걸릴 수 있다.
- 실패 시 해당 단계에 오류 표시가 나타난다 → 파일을 확인하고 다시 시도한다.

### 목록 카드 보는 법 (공통)

목록의 각 카드에는 핵심 결과가 요약되어 있다.

- **제목**: 검사 명칭(또는 파일명)
- **배지**: 공공누리 유형(제N유형) 또는 권리 요약(허용 N / 확인필요 N)
- **상태**: 완료 · 처리중 · 실패
- **하단**: 저작물/근거 건수 · 등록자 · 날짜

상단 **검색창**에 검사명·파일명을 입력해 원하는 건을 빠르게 찾을 수 있다.

## 4. 검사하기 — 결과 보는 방법

카드를 클릭하면 상세 화면이 **좌우 2단**으로 열린다.

**좌측**
- **계약서 기본정보**: 파일명(클릭 시 다운로드), 등록일, 등록자, 상태
- **공공누리 자동분류 정보**: 공공누리 마크와 함께 **제N유형**, 유형 설명, **분류 정확도(%)** 를 표시한다. 정확도는 모델이 이 유형을 확신하는 정도다.
- **계약서 추출 메타데이터**: 계약서에서 추출한 유형·계약 제목 등 요약. **[상세 보기]** 로 전체 항목 확인.

**우측**
- **저작물 메타데이터**: 좌측 **저작물 목록**에서 파일을 선택하면 해당 저작물의 20항목 메타데이터가 나타난다.

> **저작물 메타데이터 20항목**: 저작물명 · 유형 · 디지털화형태 · 설명 · 주제어 · 언어 · 제작일 · 제작자 · 저작권자 · 공동저작자 · 저작인접권자 · 공개유형 · 저작물성 · 비보호저작물 · 업무상저작물 · 상업적이용허락 · 저작재산권 · 공동저작자동의 · 유효기간 · 초상권

---

## 5. 권리추정 — 결과 보는 방법

상세 화면은 위에서 아래로 다음과 같이 구성된다.

1. **판정 요약 밴드**: `허용/확정` · `확인필요` · `없음` · `근거` 건수를 한눈에 보여준다.
2. **권리 그룹 테이블**: 권리를 그룹(저작재산권 · 이용조건 · 계약성격 · 이용범위 · 대가조건)별로 묶어 아래 항목을 표시한다.
   - **권리**: 복제권·전시권·상업적 이용 등
   - **판정**: 허용 · 금지 · 확인필요 · `-`(없음)
   - **신뢰도**: 판정을 확신하는 정도(%)
   - **근거번호**: 판정의 근거가 된 문장 번호

> 처리 중이면 진행 단계(스텝퍼)가 먼저 보이고, 완료되면 자동으로 결과로 바뀐다.

## 6. 통합검사 — 결과 보는 방법

계약서와 저작물을 함께 올리면, **한 화면에서** 메타데이터·유형·권리·근거를 모두 확인할 수 있다. 위에서 아래 순서로 본다.

1. **계약서 추출 정보**: 주제어 · 당사자 · 언어 · 계약 제목 · 기관명 · 설명 · 제작일 · 계약서 유형 등
2. **공공누리 유형**: 제N유형 배지 + 설명 + 신뢰도 + 유형 근거 문장
3. **판정 요약**: 허용/확정 · 확인필요 · 없음 · 근거
4. **원본 문서 (근거 하이라이트)**: OCR 원문 위에 판정 근거를 **색으로 표시**한다.
   - 🔵 파랑 = 유형 근거
   - 🟢 초록 = 권리 허용
   - 🔴 빨강 = 권리 금지
   
   → "왜 이 판정인가"를 원문에서 바로 확인할 수 있다.
5. **권리 그룹 테이블**: 권리별 판정 · 신뢰도 · 근거번호 (§5와 동일)
6. **저작물 메타데이터**: 업로드한 저작물별 20항목

---

## 7. 결과 이해하기 (공통)

### 7.1 공공누리 유형

| 유형 | 의미 |
|---|---|
| 제1유형 | 출처표시 |
| 제2유형 | 출처표시 + 상업적 이용금지 |
| 제3유형 | 출처표시 + 변경금지 |
| 제4유형 | 출처표시 + 상업적 이용금지 + 변경금지 |

### 7.2 권리 판정 상태

| 표시 | 의미 |
|---|---|
| 허용 | 해당 권리 이용이 허용됨 |
| 금지 | 해당 권리 이용이 금지됨 |
| 확인필요 | 자동 판정이 모호하여 담당자 확인을 권장 |
| - (없음) | 계약서에서 근거를 찾지 못함(미식별) |

### 7.3 신뢰도와 근거

- **신뢰도(%)**: 모델이 해당 판정을 확신하는 정도(0~100%). 신뢰도가 낮으면 **근거 문장을 함께 확인**한다.
- **근거번호 · 하이라이트**: 판정과 연결된 근거의 위치. 통합검사에서는 원문에 색으로 표시되어 즉시 대조할 수 있다.
- **확인필요** 항목은 자동 판정을 그대로 신뢰하기보다 원문 근거를 보고 담당자가 최종 판단한다.

## 8. 자주 묻는 질문 (FAQ)

**Q. 목록에 기존 검사 결과가 보이지 않습니다.**
A. 계정별로 접근 범위가 다릅니다. 담당 계정으로 로그인했는지 확인하세요.

**Q. 지원 파일 형식은?**
A. PDF, JPG, PNG, TIFF. 계약서는 PDF 권장.

**Q. 처리 중 상태에서 멈춰 보입니다.**
A. OCR·분석에 시간이 걸릴 수 있습니다. 상세 화면은 자동 갱신되며, 실패 표시가 뜨면 다시 시도하세요.

**Q. "계약 제목"과 "저작물명"의 차이는?**
A. **계약 제목**은 계약서가 다루는 대상의 제목(계약서 추출 정보)이고, **저작물명**은 개별 저작물 각각의 이름(저작물 메타데이터)입니다.

**Q. 신뢰도가 낮은 판정은 어떻게 하나요?**
A. 원문 근거(하이라이트·근거 문장)를 확인해 담당자가 최종 판단하세요. "확인필요" 항목도 동일합니다.
