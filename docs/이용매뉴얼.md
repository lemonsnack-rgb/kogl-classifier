# 공공저작물 권리유형 자동분류 서비스 — 이용 매뉴얼

> 본 매뉴얼은 프로토타입(`kogl-classifier`)의 사용 방법을 안내한다.
> 대상: 한국문화정보원(공공누리 운영기관) 관리자 및 검토 담당자.
> 서비스 주소: https://kogl-classifier.vercel.app
>
> - 주관: 무하유 · 참여: HM컴퍼니 · 숭실대 산학협력단 · 수요: 한국문화정보원 · 한국저작권위원회

---

## 1. 서비스 개요

계약서·저작물 파일을 업로드하면 **공공누리 유형(제1~4유형)** 과 **저작권 권리 판정(허용/금지)** 을 자동으로 분류·추정하고, 판정 근거를 원문에서 확인할 수 있는 관리자용 콘솔이다.

세 가지 메뉴로 구성된다.

| 메뉴 | 용도 | 산출물 |
|---|---|---|
| **검사하기** | 계약서·저작물로 공공누리 유형 자동 분류 | 공공누리 유형 + 근거 + 저작물 메타데이터 |
| **권리추정** | 계약서 본문에서 권리 항목별 허용/금지 추정 | 권리 판정표 + 근거 |
| **통합검사** | 메타데이터·유형·권리를 한 화면에서 제공 | 계약서 메타데이터 + 유형 + 권리 + 근거 하이라이트 |

> 유형 분류는 HM컴퍼니 유형분류 모델, 권리 추정은 HM컴퍼니 권리추정 모델, 문서 OCR·메타데이터 추출은 숭실대 API가 담당한다. 화면에 표시되는 모든 값은 해당 API의 응답값과 동일하다.

## 2. 시작하기

### 2.1 접속 및 로그인

1. 브라우저에서 https://kogl-classifier.vercel.app 접속
2. 로그인 화면에서 **이메일**과 **비밀번호** 입력 후 **[로그인]** 클릭
3. 계정이 없으면 하단 **[회원가입]** 으로 등록(관리자 승인 계정 기준)

> **중요**: 기존 검사 결과를 모두 확인하려면 **관리자 계정**으로 로그인해야 한다. 일반 계정은 본인이 등록한 검사만 보인다.

### 2.2 화면 구성

- **좌측 사이드바**: 검사하기 · 권리추정 · 통합검사 메뉴, 바로가기(공공누리·한국문화정보원), 마이페이지, 로그아웃
- **본문 영역**: 선택한 메뉴의 목록/업로드/결과 화면

세 메뉴 모두 동일한 화면 구성(상단 제목·설명·검색, 카드형 목록)을 사용한다.

## 3. 검사하기 (공공누리 유형 분류)

### 3.1 목록 화면

- 상단에 **검사명·파일명 검색창**이 있다. 키워드 입력 후 **[검색]**.
- 카드마다 **공공누리 유형 배지(제N유형/미분류)**, **검사 상태**, **저작물 건수**, **등록자 · 등록일** 이 표시된다.
- 첫 번째 **[새 검사 시작]** 카드로 신규 검사를 등록한다.

### 3.2 새 검사 등록 (3단계)

**1단계 — 기본정보 및 계약서**
1. **검사 명칭** 입력(필수)
2. **문서 유형** 선택(계약서 / 동의서 / 저작재산권 양도동의서 / 공공저작물 자유이용허락 동의서 / 기타문서)
3. 기관 자체 제작물이면 **[기관 자체 제작물입니다]** 체크
4. **계약서 파일** 업로드(필수, 드래그&드롭 또는 클릭). 동의서는 선택
5. **[다음]**

**2단계 — 저작물 업로드**
1. 분류할 **저작물 파일**을 업로드(복수 선택 가능)
2. 업로드된 파일 목록 확인 후 **[다음]**

**3단계 — 확인 및 검사**
1. 검사 명칭·문서 유형·계약서/동의서·저작물 목록을 확인
2. **[검사하기]** 클릭 → 처리 시작(목록으로 이동)

### 3.3 결과 화면 (2단 구성)

- **좌측**
  - **계약서 기본정보**: 파일명(다운로드 가능), 등록일, 등록자, 상태, 자체제작 여부
  - **공공누리 자동분류 정보**: 공공누리 마크 + **제N유형** + 유형 설명 + **분류 정확도(%)**. 필요 시 **[수정]** 으로 관리자 보정
  - **계약서 추출 메타데이터**: 유형, **계약 제목** 등 (상세는 [상세 보기])
  - **저작물 목록**: 업로드된 저작물 파일 목록
- **우측**
  - **저작물 메타데이터**: 좌측 목록에서 저작물을 선택하면 20항목 메타데이터 표시

> **저작물 메타데이터 20항목**: 저작물명·유형·디지털화형태·설명·주제어·언어·제작일·제작자·저작권자·공동저작자·저작인접권자·공개유형·저작물성·비보호저작물·업무상저작물·상업적이용허락·저작재산권·공동저작자동의·유효기간·초상권

## 4. 권리추정

### 4.1 목록 화면

- 카드마다 **권리 요약 배지(허용 N / 확인필요 N)**, **상태(완료/실패/처리중)**, **근거 건수**, **날짜** 표시.
- 상단 검색창으로 검사명·파일명 검색.

### 4.2 새 권리추정 등록

1. **[새 권리추정]** 카드 클릭
2. 입력 방식 선택 — **[파일 업로드]** 또는 **[텍스트 입력]**
3. **문서 유형** 선택
4. (파일) **검사 명칭** 입력 + **PDF 파일** 선택 / (텍스트) **계약서 본문** 붙여넣기
5. **[권리추정 실행]**

> 파일 업로드 시 숭실대 API로 OCR·메타데이터를 먼저 추출한 뒤 권리를 추정한다. 텍스트 입력 시 OCR을 건너뛰고 바로 추정한다.

### 4.3 결과 화면

- **요약 밴드**: 허용/확정 · 확인필요 · 없음 · 근거 건수
- **권리 그룹 테이블**: 저작재산권·이용조건·계약성격·이용범위·대가조건 그룹별로 **권리 / 판정(허용·금지·확인필요) / 신뢰도 / 근거번호** 표시
- 처리 중이면 **진행 단계 표시(스텝퍼)** 가 나타나고, 완료되면 자동으로 결과로 전환된다.

## 5. 통합검사

계약서와 저작물을 함께 올려 **메타데이터 · 공공누리 유형 · 권리 판정 · 근거 하이라이트**를 한 화면에서 확인한다.

### 5.1 새 통합검사 등록

1. **[새 통합 검사]** 카드 클릭
2. **문서 유형** 선택
3. **검사 명칭** 입력(필수)
4. **계약서 파일** 업로드(필수) — 계약서 추출정보·공공누리 유형·권리 판정에 사용
5. **저작물 파일** 업로드(선택, 복수) — 저작물별 20항목 메타데이터 추출에 사용
6. **[통합 검사 실행]**

### 5.2 결과 화면 (한 화면 통합)

위에서 아래로 다음 순서로 표시된다.

1. **계약서 추출 정보**: 주제어·당사자·언어·계약 제목·기관명·설명·제작일·계약서 유형 등
2. **공공누리 유형 (유형분류 모델)**: 제N유형 배지 + 설명 + 신뢰도 + 유형 근거 문장
3. **판정 요약**: 허용/확정 · 확인필요 · 없음 · 근거
4. **원본 문서 (근거 하이라이트)**: OCR 원문 위에 근거를 색으로 표시
   - 파랑 = 유형 근거 · 초록 = 권리 허용 · 빨강 = 권리 금지
5. **권리 그룹 테이블**: 권리별 판정·신뢰도·근거번호
6. **저작물 메타데이터**: 업로드한 저작물별 20항목

## 6. 결과 이해하기

### 6.1 공공누리 유형

| 유형 | 의미 |
|---|---|
| 제1유형 | 출처표시 |
| 제2유형 | 출처표시 + 상업적 이용금지 |
| 제3유형 | 출처표시 + 변경금지 |
| 제4유형 | 출처표시 + 상업적 이용금지 + 변경금지 |

### 6.2 권리 판정 상태

| 표시 | 의미 |
|---|---|
| 허용 | 해당 권리 이용이 허용됨 |
| 금지 | 해당 권리 이용이 금지됨 |
| 확인필요 | 자동 판정이 모호하여 관리자 확인 권장 |
| - (없음) | 계약서에서 근거를 찾지 못함(미식별) |

### 6.3 신뢰도와 근거

- **신뢰도(%)**: 모델이 해당 판정을 확신하는 정도(0~100%). 낮으면 근거를 함께 확인한다.
- **근거번호**: 판정과 연결된 근거 문장 번호. 통합검사에서는 원문 하이라이트로 위치를 함께 보여준다.

### 6.4 처리 진행 단계

검사는 다음 단계로 진행되며, 상세 화면에서 실시간으로 표시된다.

```
업로드 → OCR·메타데이터 → (공공누리 유형) → 권리 판정 → 완료
```

- 처리 중에는 목록/상세가 자동 갱신된다.
- 실패 시 해당 단계에 오류 표시가 나타난다 → 파일 상태를 확인하고 재시도한다.

## 7. 자주 묻는 질문 (FAQ)

**Q. 목록에 기존 검사 결과가 보이지 않습니다.**
A. 일반 계정은 본인이 등록한 검사만 보입니다. 전체 결과 확인은 **관리자 계정**으로 로그인하세요.

**Q. 지원 파일 형식은?**
A. PDF, JPG, PNG, TIFF. 계약서는 PDF 권장.

**Q. 처리 중 상태에서 멈춰 보입니다.**
A. OCR·모델 분석에 시간이 걸릴 수 있습니다(문서 크기에 따라 수십 초~수 분). 상세 화면은 자동 갱신되며, 실패 표시가 뜨면 재시도하세요.

**Q. 표시된 유형/권리를 수정할 수 있나요?**
A. 검사하기 상세의 **[수정]** 으로 공공누리 유형·메타데이터를 관리자가 보정할 수 있습니다. 원본 자동 판정값은 API 응답값과 동일하게 저장됩니다.

**Q. 통합검사의 "계약 제목"은 무엇인가요?**
A. 계약서 추출 정보에 표시되는, 계약서가 다루는 대상의 제목입니다. 아래 "저작물 메타데이터"의 개별 저작물명과는 구분됩니다.
