# 공공누리 신유형(제0유형·AI유형) 판정 설계 메모

작성일: 2026-07-21
대상: kogl-classifier 프로토타입
전제: **본 메모는 설계안이며, 코드 수정은 포함하지 않는다.**

---

## 1. 배경

공공누리 개정('26.1.28 공고)으로 신유형 2종이 신설되었다.

- **제0유형(자유이용, 신설)**: 조건 없이 자유 이용. 출처명시 불요, 상업적 이용 가능, 변경 가능.
- **AI유형(인공지능 학습 가능, 신설)**: AI 학습 목적에 한정한 자유 이용. **단독 부착 불가, 제1~4유형과 병행 표기.**
- 기존 만료 공공저작물 표시는 폐지 → 제0유형으로 전환.

현재 프로토타입은 **유형1~4만** 판정한다. 담당자 의견은 "신유형은 기존 판단 근거를 그대로 쓸 수 있다"였고, 조사 결과 **대체로 타당하나 판정 규칙 추가가 필요**하다.

---

## 2. 신유형 판정 기준 (가이드라인 요지)

제0유형은 **이용허락 범위(상업·변경)가 아니라 권리귀속**으로 결정된다.

| 공공저작물 구분 | 권장 유형 | 판정 조건 |
|---|---|---|
| 업무상저작물 (기관이 저작권 원시적 전부 보유) | **제0유형** | 제3자 저작물·초상 미포함 |
| 만료 저작물 (사후 70년 / 공표 70년 경과) | **제0유형** | 보호기간 만료 |
| 업무상저작물에 제3자 저작물·초상 포함 | 제1~4유형 + AI유형 | 권리자 동의 필요 |
| 제3자 저작물 (계약으로 권리 취득) | 제1~4유형 + AI유형 | 권리자 동의 필요 |

**AI유형**은 콘텐츠에서 판정하는 값이 아니라, 제1~4유형에 **권리자 동의를 전제로 병행 부착**하는 태그다. 별도 모델 학습이 필요 없다.

### 공공누리 적용 제외 대상 (참고, 지침 제11조 제2항)
국가안전보장 정보, 개인 사생활·사업상 비밀, 타 법률상 공개제한 정보, 국유·공유재산으로 관리되는 등록저작물, 제3자 미동의 → 공공누리 적용 자체가 제외.

## 3. 핵심 문제: 유형1~4 축만으로는 제0유형을 못 가른다

현재 HMC 유형 모델은 계약서 텍스트에서 **상업적이용·변경** 두 축을 읽어 유형1~4를 판정한다.

| 유형 | 출처표시 | 상업 | 변경 |
|---|---|---|---|
| 제0유형 | **불요** | O | O |
| 제1유형 | 필요 | O | O |
| 제2유형 | 필요 | X | O |
| 제3유형 | 필요 | O | X |
| 제4유형 | 필요 | X | X |

제0유형과 제1유형은 상업·변경 축이 동일(O·O)하고 **차이는 출처표시뿐**이다. 따라서 HMC 출력만으로는 제0유형을 도출할 수 없고, **권리귀속(업무상저작물/만료/제3자 포함) 판단이 추가로 필요**하다.

---

## 4. 결론: HMC API 수정 없이 서비스 계층에서 해결 가능

판정에 필요한 신호가 이미 서비스로 들어오기 때문이다.

파이프라인(`src/app/api/pipeline/process/route.ts`)은 두 API 결과를 **서비스가 직접 조립**한다.

```
[3단계] 숭실대(SSU) API  → OCR + 메타데이터(제0유형 판정 근거)
[4단계] HMC API          → 유형1~4 (상업·변경 축)
        서비스가 TYPE_MAP 매핑 → gongnuri_type DB 저장  ← 여기에 규칙 삽입
```

SSU 메타 추출은 이미 제0유형 판정 근거를 산출한다(`pipeline-server/main.py`의 `map_ssu_to_work_fields`, `src/types/index.ts`의 Work 20항목).

| 판정 근거 | 필드 | 출처 |
|---|---|---|
| 업무상저작물 | `work_for_hire` (15항목) | SSU 메타 |
| 비보호저작물 | `non_protected_work` (14항목) | SSU 메타 |
| 만료/유효기간 | `validity_period` / `copyright_period` (19항목) | SSU 메타 |
| 초상권(제3자 초상 포함) | `portrait_rights` (20항목) | SSU 메타 |
| 저작재산권 보유범위 | `property_rights` (17항목) | SSU 메타 |
| 공동저작자 동의 | `co_author_consent` (18항목) | SSU 메타 |
| 출처표시 | `ATTRIBUTION_REQUIRED` | 권리추정 모델(rights-server) |
| 상업·변경 | HMC 유형1~4 | HMC API |

→ 새로 학습해서 뽑아야 할 신호가 아니라 **이미 추출 중인 필드를 조합하는 후처리 규칙**만 추가하면 된다.

## 5. 판정 규칙 (의사결정 트리)

서비스 후처리에서 다음 순서로 적용한다.

```
0. 공공누리 적용 제외 대상인가? (국가안전보장/사생활/비밀 등)
   └ 예 → 공공누리 적용 제외 (판정 중단)

1. 보호기간 만료(비보호) 저작물인가?
   (non_protected_work=예 OR validity_period 만료)
   └ 예 → 제0유형

2. 업무상저작물이면서 제3자 권리·초상 미포함인가?
   (work_for_hire=예 AND portrait_rights 없음 AND 제3자 저작물 미포함)
   └ 예 → 제0유형

3. 그 외 (계약 취득 / 제3자 포함 / 권리 일부 보유)
   └ HMC 유형1~4 결과 유지
      └ 권리자 동의 확보됨(co_author_consent 등) → 유형N + AI유형 병행
      └ 동의 없음/미확인 → 유형N 단독 (AI유형 부착 보류, 검토 대상)
```

### 규칙 보강 원칙
- **AI유형은 제0유형과 병행하지 않는다** (제0유형은 그 자체로 AI 학습 자유). 제1~4유형에만 병행.
- 제0유형으로 override할 때, **근거 표시**는 HMC 근거문장이 아니라 판정에 사용한 메타 필드(예: "업무상저작물=예, 초상권 없음")로 생성한다.
- 자동 판정 신뢰도가 낮거나 필드가 비어 있으면 `review_required`(담당자 검토) 상태로 보낸다.

## 6. 구현 위치 및 스키마 영향 (설계만, 미구현)

### 6.1 후처리 삽입 위치
- `src/app/api/pipeline/process/route.ts` 4단계(HMC 호출) 직후, `gongnuri_type` DB 저장 직전.
- SSU 메타(`consolidated_metadata`)와 HMC 결과가 모두 확보된 지점이므로 여기서 규칙 함수를 호출해 최종 유형을 확정한다.

### 6.2 타입/상수 변경
- `src/types/index.ts`
  - `KoglType`에 `"KOGL-0"` 추가.
  - `KOGL_TYPES`에 제0유형 메타 추가(label "제0유형", description "자유이용", color).
  - AI유형은 유형과 **직교하는 부가 속성**이므로 별도 필드로 설계(예: `ai_type_applied: boolean`). 유형 enum에 섞지 않는다.
- `src/lib/api/classifier.ts`
  - `mapHMCToKoglType`은 그대로 두고(HMC는 여전히 유형1~4만 반환), 신유형 확정은 별도 후처리 함수로 분리.

### 6.3 DB
- `contracts.gongnuri_type`에 `KOGL-0` 값 허용.
- AI유형 병행 여부 저장 컬럼(예: `ai_type_applied`) 신설 검토.

### 6.4 UI
- 결과 화면(`RightsResultView.tsx`, 유형 뱃지 사용처)에 제0유형 뱃지, AI유형 병행 뱃지 표기 추가.

## 7. 예외 처리: 업무상저작물은 계약서가 없을 수 있다

제0유형 주 대상인 업무상저작물은 **계약서가 아예 없는 경우**가 많다(직원이 작성한 보도자료·보고서 등). 이 경우 HMC(계약서 기반)에 넣을 텍스트가 없다.

→ 후처리에서 분기 필요:
- 계약서 없음 + 업무상저작물/만료 → **HMC 스킵, 메타만으로 제0유형 판정**
- 계약서 있음 → 기존 흐름(HMC 유형1~4) + 위 규칙

## 8. 업체 요청 필요 여부

| 대상 | 요청 필요? | 사유 |
|---|---|---|
| HMC(유형 모델) | **불필요** | 규칙 기반 후처리로 해결. 모델이 제0유형을 네이티브 출력하게 하려면 재라벨링·재학습이 필요하나 불필요. |
| 숭실대(SSU, 메타 추출) | 조건부 | `work_for_hire`·`validity_period`·`portrait_rights` 추출 정확도가 낮으면 품질 개선 요청. 판정 로직이 아닌 **입력 데이터 품질** 이슈. |

---

## 9. 진행 전 확인 필요 사항

1. SSU 메타의 `work_for_hire` / `validity_period` / `portrait_rights` 필드가 실제 문서에서 **얼마나 정확히 채워지는가** (샘플 검증 필요).
2. 만료 판정 기준을 필드값(문자열)에서 어떻게 파싱할지 — 공표일/사후 70년 계산 로직 필요 여부.
3. 권리자 동의 상태(`co_author_consent` 등)로 AI유형 병행 여부를 자동 판정할지, 담당자 수동 확정으로 둘지.
4. 제0유형 override 시 담당자 검토를 항상 거칠지(초기에는 권장), 신뢰도 임계값 이상만 자동 확정할지.
